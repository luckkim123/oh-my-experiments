## 2. Component design

### 2.1 `omx_core/lock.py`

```
with_file_lock(lock_path, fn, *, timeout_s=5.0, retry_s=0.05)   # fcntl mutex (extracted from with_wiki_lock verbatim, WikiError → OmxError)
acquire_run_lease(paths, run_id, *, session_id, now_iso, stale_hours=2) -> dict   # O_EXCL create; on EEXIST: reap-or-fail
release_run_lease(paths, run_id) -> {"released": bool}           # unconditional; releasing an unheld lease returns {"released": false}
read_run_lease(paths, run_id) -> dict | None                     # pure read for loop-status
```

- Lease payload: `{"session_id": str|None, "armed_at": iso, "armed_by_pid": int}` written via O_EXCL (`os.open(..., O_CREAT|O_EXCL|O_WRONLY)`) — creation is the atomic claim; no `.tmp` dance (a lease must not be rename-replaceable). `armed_by_pid` is recorded for forensics only; **no code path reads it for decisions** (one-shot-CLI model, D-R4-3).
- `acquire_run_lease` mkdirs `runs/<run_id>/` first (arm can precede tree scaffolding; ENOENT is not a claim failure).
- Reap condition: lease age > `stale_hours` (from `armed_at`; mtime fallback when the JSON is corrupt — a corrupt lease is a stale-candidate, never insta-reaped). Reaping unlinks and re-attempts O_EXCL once; **ENOENT on the unlink means another reaper won — also retry the O_EXCL once**; losing the re-create race loud-fails as "run owned". A young lease (age ≤ stale_hours) with a different `session_id` → loud-fail naming the owning session and `armed_at`.
- `wiki/storage.py::with_wiki_lock` becomes `return with_file_lock(paths.wiki_lock(), fn, ...)` re-raising as `WikiError` (existing wiki tests keep passing — the extraction is behavior-preserving).
- New paths getters: `omx_paths.loop_lock(run_id)` → `runs/<run_id>/.loop-lock`, `omx_paths.state_lock()` → `.omx/state/.state-lock`, `omx_paths.loop_marker_json(run_id)` (file name `loop-status.json`).
- `LOCK_STALE_HOURS = 2` named constant; profile override key `lock_stale_hours`.

### 2.2 State-mutation locking + lease wiring

- `arm_loop(paths, *, run_id, now_iso, max_runtime_s, hard_cap=50, session_id=None)`: the entire load→check-armed→write runs inside `with_file_lock(paths.state_lock(), ...)`; after the armed check passes, `acquire_run_lease(...)` — lease failure (another session's young lease) loud-fails the arm naming the owner. Lease acquired but the state write fails → lease released (try/finally). Arm also unlinks any stale `loop-status.json` marker for this run (D-R4-8).
- `disarm_loop(paths, *, reason="cancel", now_iso=None)`: same state lock; writes the completion marker for `env["run_id"]` (aware-UTC `ended_at` computed internally when `now_iso` is None — the handler path), releases the lease **unconditionally** (`release_run_lease` has no owner check: whichever process disarms is authoritatively ending the loop — this is what lets the gate's self-disarm branches clean up, critic C2), then nulls `active_loop`. Idempotent as today (`{was_armed: false}` when unarmed).
- `loop_gate`: the load→adopt/increment→save block runs under the same `with_file_lock`; a lock timeout inside the handler → the outer `except` fail-opens (a contended gate never traps the session).
- CLI plumbing: `loop-arm` gains `--session-id` (the SKILL passes the `omx session-id` value it already resolves, `exp-loop/SKILL.md:41-44`); `run-record` gains `--session-id`; `loop-disarm` needs none (release is unconditional).

### 2.3 Ledger verbs — `run-seed`, `run-record`

- `omx run-seed --run-id <r> --baseline-commit <sha> --keep-policy <p> [--root]` → `seed_ledger` verbatim; loud-fails if the ledger already exists (seeding is once).
- `omx run-record --run-id <r> --iteration N --decision keep|discard|ambiguous|bootstrap --candidate-checkpoint <path> --candidate-commit <sha> --description <s> [--session-id <sid>] [--cwd <project>] [--eval-json <path>] [--no-staleness-check] [--root]`:
  1. **lease assertion**: a lease exists AND carries a `session_id` AND the caller passed `--session-id` AND they differ → rc 2 loud-fail ("run <id> is owned by loop session <sid> (armed <ts>); disarm it or pass the owning --session-id"). Caller passed no `--session-id` while a lease exists → stderr warning (ownership unverifiable), proceed. No lease → proceed silently (single-shot record outside a loop is legitimate).
  2. **staleness check** (D-R4-6): with `--cwd` a git repo — resolve `base = ledger.last_kept_commit or ledger.baseline_commit`; `git merge-base --is-ancestor <base> <candidate-commit>` (the `run(check=False)` idiom); non-ancestor → rc 2 loud-fail naming both shas and the `--no-staleness-check` escape; `--cwd` missing / not a repo / shas unresolvable → stderr warning `staleness check skipped (<why>)`, proceed.
  3. `record_iteration(...)` (unchanged core);
  4. prints the updated pointer trio `{last_kept_commit, last_kept_checkpoint, last_kept_score}` + the appended entry.
- `--eval-json` (optional): path to a saved `omx eval` output; when given, its `decision` block is embedded as the `evaluator` field — keeps `run-record` consumable both by the loop (which just ran `eval`) and by hand.
- exp-loop SKILL step 4 rewrites from "record through the ledger writer" (prose, `SKILL.md:112`) to the concrete `omx run-seed` (first iteration) / `omx run-record` commands — which also puts them under the #25 contract test at last.

### 2.4 Evaluator fault taxonomy + wiki capture

- `evaluator.py`: each of the four error branches sets `fault_class`: `"timeout"` (:87-94), `"nonzero_exit"` (:97-101, which also gains `parse_error: "evaluator exited non-zero"` for shape uniformity), `"empty_stdout"` (:103-108), `"unparseable"` (:109-116). Success records carry no `fault_class` key.
- `decision.py` error branch note becomes `"candidate discarded because evaluator errored (<fault_class>)"` — `decision_reason` stays `"evaluator error"` (downstream stability).
- `_cmd_eval`: after printing, when `status == "error"` and `--root` was given — lazy-import wiki ingest and append-merge a stub page: slug `evaluator_fault_<fault_class>` (title-derived; wiki slugs are underscore-constrained), category `debugging`, confidence `low`, tags `[auto-captured, evaluator-fault, <fault_class>]`, body = command, exit_code, parse_error, ran_at, cwd. Any exception → stderr warning only (grading never breaks on knowledge plumbing). rc semantics unchanged (rc 1 evaluator-broke).
- `omx loop-health` (§2.6) counts a fault streak from ledger entries whose embedded evaluator record has `status == "error"`.

### 2.5 Completion marker

- `loop.py::mark_loop_done(paths, run_id, *, reason, summary, now_iso)` → atomic write of `runs/<run_id>/loop-status.json` `{"schema_version": 1, "phase": "done", "reason", "summary", "ended_at"}`. Overwrite allowed (a re-run of a run id refreshes the marker; history lives in the ledger, not the marker). `arm_loop` unlinks it (a freshly-armed loop must not read `done`).
- `disarm_loop` calls it with the disarm reason (summary = `"iteration <n>"`) whenever an envelope was armed. The unarmed CLI verb `omx loop-mark-done --run-id --reason done|deadline|cancel|error --summary <s>` covers single-pass (never-armed) flows.
- `omx loop-status --run-id` gains `phase`: `done` (marker present) → else `running` (armed for this run, deadline not passed) → else `died` (armed with deadline passed and no marker, or an orphan lease with no armed envelope naming the run) → else `idle`. The marker is read fresh; no caching.

### 2.6 `loop_health` + circuit wiring

- `ledger.py::read_run_ledger(paths, run_id) -> dict` — loud-fail on absent/corrupt (the read half D-R4-2's writers finally justify).
- `loop.py::loop_health(ledger, *, plateau_discards=5, fault_streak=3) -> {"consecutive_discards", "consecutive_faults", "iterations", "last_kept_iteration", "plateau_tripped", "fault_tripped"}` — pure function over `entries` tail (streaks reset on any `keep`); no clock math (clock-safe by construction).
- `omx loop-health --run-id [--root]` — resolves thresholds from profile keys `plateau_discards`/`fault_streak` (fallback constants), prints the health JSON; **rc 2 when either circuit is tripped** (message names the streak and the knob), rc 0 otherwise. **This verb + SKILL step 4.5 is the authoritative stop path** (D-R4-4).
- `loop_gate` cascade inserts after the hard_cap branch, as the best-effort backstop:

```
circuit tripped (loop_health on env["run_id"])  -> disarm(reason=plateau|fault_circuit), allow
```

wrapped in its own try/except → on any circuit-evaluation error (including a missing ledger: a loop that never recorded), skip the branch. For the first iterations the branch is structurally a no-op (fewer entries than any threshold) — expected, not a defect; the residual (a ledger-read failure silently disables the backstop, preserving fail-open) is documented in §4.
- exp-loop SKILL gains step 4.5: `omx loop-health --run-id <r> --root <root>` — rc 2 → stop the loop (`omx loop-disarm --reason plateau|fault_circuit`), report the streak to the user.

### 2.7 `loop-status --all`

`omx loop-status --all [--root]` (mutually exclusive with `--run-id`): glob `runs/*/`, and for each run id emit `{run_id, phase, lease: {session_id, armed_at}|null, marker: {reason, ended_at}|null, armed: bool}` (armed = the root's single envelope names this run). Output `{"runs": [...], "armed_run": <id>|null}`. Pure read; corrupt per-run files degrade to `"phase": "unknown"` with a stderr warning, never fatal (wiki corrupt-page-tolerance idiom).

### 2.8 `revert-config`

- `omx_core/revert.py::plan_revert(cwd, sha, protected) -> {"would_revert": [...], "skipped_allowlist": [...]}` — `git diff --name-only <sha>` via `subprocess.run(..., check=False)` (the `gc.py:116-118` idiom; a non-zero rc or OSError converts to loud-fail at the verb layer). `protected` = the path prefixes never reverted: `.omx/` (relative to cwd) and, when the resolved `--root`'s `.omx` tree lies inside `--cwd`, that tree's relative prefix too. `apply_revert(cwd, sha, paths)` — `git checkout <sha> -- <paths>` with `check=True` (loud by design).
- `omx revert-config --cwd <project> --run-id <r> [--to baseline|last-kept|<sha>] [--i-approve-revert] [--root]`:
