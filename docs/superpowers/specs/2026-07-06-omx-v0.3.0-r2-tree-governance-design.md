## 2. Component design

### 2.1 tree.yaml — the typed-section schema

Core (in `tree.py`) understands exactly six section types; every value is instance-side. The instance below is what `tree-codify` should produce for the reference workspace (values traced to `paths.py`); the shipped default follows.

```yaml
version: 1                       # int, required; loud-fail on unknown version
trees:                           # 1+ roles; 'index' REQUIRED, others optional
  index:                         # light tree: manifests, config, eval, analysis
    root: experiments            # path relative to the omx root (the dir holding .omx/); absolute allowed
    levels: [framework, exp, "group?"]   # named hierarchy above run dirs; '?' = optional level
  data:                          # heavy tree; ABSENT = single-tree layout
    root: logs/rsl_rl
    levels: [exp]
run_id:
  grammar: "<label>[_<tag>]_<ts>"        # v1 supports exactly this field family (label-before-date)
  ts_format: "%y%m%d_%H%M%S"             # strftime; compiled to a digit regex for parsing
  tag: required                          # required | optional  (F8: required + absent = violation)
links:                           # named directory entries with semantics
  train:    {kind: data_pointer, required: true}   # the run-dir entry that points into the data tree
  latest:   {kind: alias, scope: group}            # symlink at the scope level pointing to one run dir
  baseline: {kind: alias, scope: group}
run_dir:
  requires: [manifest.json]              # files every run dir must contain (audit: warn)
  entries: [config, analysis]            # optional allowlist of non-link run-dir entries (absent key = no check)
  eval_pattern: "eval/<mode>_<ts>"       # placeholders <mode>, <ts>
  eval_modes: [static, periodic, segmented, sudden]
  deprecated:                            # declared anti-patterns (F3: the silent-fallback residue)
    - {pattern: "eval_dr", message: "pre-standard fallback dir; artifacts belong in eval/<mode>_<ts>/"}
    - {pattern: "eval_dr_robustness", message: "pre-standard fallback dir"}
walk:
  ignore: ["legacy", "_*"]               # segments skipped everywhere (plus: symlinks are never runs)
```

Shipped default instance (written by `omx init` when tree.yaml is absent; §10.1-shaped, zero workspace vocabulary):

```yaml
version: 1
trees:
  index: {root: experiments, levels: []}
run_id:
  grammar: "<label>[_<tag>]_<ts>"
  ts_format: "%y%m%d_%H%M%S"
  tag: optional
links:
  latest: {kind: alias, scope: root}
run_dir:
  requires: [manifest.json]
  eval_pattern: "eval/<mode>_<ts>"
  eval_modes: [static]
walk:
  ignore: ["legacy", "_*"]
```

**run_id grammar semantics.** v1 supports the single grammar family `<label>[_<tag>]_<ts>` with fixed charsets: label = `[a-z0-9][a-z0-9-]*` (dash allowed, **underscore forbidden**), tag = `[A-Za-z0-9][A-Za-z0-9_-]*` (underscores allowed), ts = the digit pattern compiled from `ts_format` (`%y/%m/%d/%H/%M/%S` → `\d{2}`, `%Y` → `\d{4}`; any non-digit-producing token → loud-fail at schema load). Parsing: split the leaf on `_`; the trailing `ts_format.count('_')+1` fields must match the ts regex (and `strptime`); the first remaining field is the label; everything between is the tag (re-joined on `_`). `tag: required` + empty middle = F8 violation. This split is deterministic **under the dash-only-label assumption**. The reference writer's registered vocabulary satisfies it (`trpo`, `ppo-enc`, `trpo-noipo`; `paths.py:57-64`); its *fallback* path does not — `task_short` slugifies unregistered task IDs with underscores (`paths.py:84`), and such a leaf parses with the label's underscore tail mis-attributed to the tag. The consequence is bounded: the split never crashes, overall shape validation (T7) is unaffected, and only the F8 tag-presence semantics weaken for those leaves. v1 documents this as a known limitation (repeated in the header comment codify writes into tree.yaml); a declared label vocabulary restoring exactness is earmarked for a later grammar version, not R2 (the reference project's registered vocab is dash-only — YAGNI). The `make_run_id` date-prefix-strip rule (`paths.py:37,106-110`) is a *writer* concern and stays workspace-side; the schema only validates shape.

**Semantics of the other sections.** `trees.*.root` resolves relative to the omx root (the directory containing `.omx/` — ties into §2.8); a run dir lives at exactly `root/<levels...>/<run_id>` with optional levels omittable (F7 = a run-shaped dir at any other depth). `links.kind: data_pointer` = an entry inside a run dir that must exist (`required: true`) and, when it is a symlink and a `data` tree is declared, must resolve inside `data.root` (F1/F5 discipline). `links.kind: alias` = a symlink at its `scope` level (root | one of the declared level names) pointing at a run dir; the walker never counts symlinks as runs (`find_runs` skip rule, `paths.py:184-207`). `walk.ignore` reimplements the `legacy` / `_`-prefix skip as instance data.

**Run-dir detection predicate** (extends `paths.py:167-170`): a directory is detected as a run when (a) any `run_dir.requires` file is present, OR (b) any declared `data_pointer` entry exists, OR (c) its name parses under the run_id grammar AND it sits at a declared run depth. Clause (c) keeps the walker functional on trees whose runs predate manifests and declare no data pointer (the shipped default instance's only link is an alias), and makes T8 meaningful: detection and the requires-check are deliberately different tests — T8 fires on a run detected via (b)/(c) that is missing a required file. A grammar-valid leaf at an *undeclared* depth is never silently adopted as a run; it is exactly the T6 violation.

**Schema self-validation** (`tree.py`, loud-fail rc2 via callers): unknown top-level or per-section keys rejected (typo guard); `index` tree required; level names and eval modes must be lowercase tokens; `ts_format` must compile to a fixed-width digit pattern; alias scopes must name `root` or a declared level; deprecated patterns must be non-empty strings.

### 2.2 `omx tree-codify` (+ init emission, exp-init topic, sync-profile projection)

- `omx tree-codify [--root <r>] [--index-root <p>] [--data-root <p>] [--force]` — infer an instance from an existing tree and atomic-write `.omx/profile/tree.yaml` with `pending_approval: true`, printing an inference report as the stdout JSON. Confidence is reported as **matched/sampled evidence pairs** (so "1/1" is distinguishable from "180/180" — one run gives ratio 1.0 with no discriminating power). Loud-fail rc2 if tree.yaml already exists (`--force` to regenerate). **Zero detected runs → rc2** with guidance ("no runs to infer from; `omx init` writes the generic default instance") — codify is census-based and refuses to guess from an empty tree; the fresh-project path is `omx init`, not codify.
- Inference: index root = `--index-root` hint, else `experiments/` if it exists under the omx root (generic conventional default), else loud-fail asking for the hint. Run-dir detection = contains `manifest.json` OR a symlink entry pointing outside the index tree (data-pointer candidate). Levels = depth census from index root to detected runs (optional level where depth varies); level *names* default to `level1..levelN` — names are cosmetic (audit messages only) and the exp-init interview offers renames. `ts_format` = best-match over a candidate list (`%y%m%d_%H%M%S`, `%Y%m%d_%H%M%S`, `%Y-%m-%d_%H-%M-%S`) against run-leaf tails; confidence = match ratio. `tag` = `required` when ≥90% of runs carry one. Links = symlink-name census (inside run dirs → data_pointer; at level dirs targeting run dirs → alias). `data.root` = common ancestor of data-pointer targets. `eval_modes` = distinct `<mode>` prefixes under matched eval dirs. `deprecated` is never inferred (human-declared; the exp-init topic offers the two known fallback names as a suggestion).
- `omx init` gains one additive behavior: when `.omx/profile/tree.yaml` is absent, emit the shipped default instance (marked `pending_approval: true` like the other profile files). Existing projects are untouched until they opt in via codify.
- exp-init SKILL.md gains one interview topic: if the chosen output_root already contains runs, offer `tree-codify` (the skill calls the verb; it never writes tree.yaml itself).
- `wiki sync-profile`: add the literal filename `'tree.yaml'` to `_PROJECTED` (`sync.py:18` — entries are bare filenames joined as `profile_dir / name`, `sync.py:44`); that single addition puts tree.yaml into the staleness max-mtime AND the projection (a 3-line summary: tree roles + roots, level names, link names). I-1 pattern: content source and staleness set change in the same commit.

### 2.3 `omx tree-audit`

`omx tree-audit [--root <r>] [--strict]` — load tree.yaml (absent → rc2 naming `omx init` / `tree-codify`), walk every declared tree, report violations as JSON `{ok, counts: {error, warn}, violations: [{check, severity, path, message}]}`. Default rc0 report-only (lint family, INV-1); `--strict` → rc2 when errors ≥ 1. Checks, mapped to the soak friction seeds:

| Check | Severity | Seed |
|:--|:--|:--|
| T1 required data_pointer entry missing from a run dir | error | F1 |
| T2 data_pointer symlink dangling | error | F5 |
| T3 data_pointer target escapes the declared data tree | error | F1 |
| T4 alias symlink dangling | error | F5 |
| T5 alias target not a grammar-valid run dir, or outside its scope | error | — |
| T6 run-shaped dir at undeclared depth (missing/extra level segment) | error | F7 |
| T7 run_id grammar violation, incl. missing tag under `tag: required` | error | F8 |
| T8 `run_dir.requires` file missing | warn | — |
| T9 undeclared run-dir entry (only when `entries` allowlist present) | warn | — |
| T10 `deprecated` pattern hit | error (carries the declared message) | F3 |
| T11 eval leaf not matching `eval_pattern` × `eval_modes` | warn | — |
| T12 mirror coherence: data-tree run with no index counterpart (via the data_pointer target set), or index run whose pointer set misses it | warn | — |

Detection uses the §2.1 three-clause predicate, so T8 can actually fire (a manifest-less run is still detected via clauses (b)/(c)); a grammar-valid leaf at an undeclared depth surfaces as T6, never as a silently adopted run.

F4 (same-second collision) is deliberately NOT an audit check — a collision merges into one leaf and is not reliably detectable post-hoc; prevention lands in `tree-scaffold` (below). exp-analyze's When-done gains one report-only `tree-audit` line (skill text).

### 2.4 `omx tree-scaffold`

`omx tree-scaffold --run-id <id> [--exp <name>] [--group <g>] [--data-dir <p>] [--root <r>]`, or `--eval-for <run_spec> --mode <m>`:

- Run form: validate `run_id` against the grammar (rc2 on violation — including missing tag under `tag: required`, making F8 preventable at mint time, the workspace `train.py:234-240` WARN promoted to a refusal *in the verb*); create the level dirs + run dir + `run_dir.entries` dirs; when `links.train` is declared and `--data-dir` given, create the relative data-pointer symlink (`emit_run_manifest`'s relpath pattern, `paths.py:508-514`). No manifest is fabricated (that is the training pipeline's job) and nothing is ever launched (D4-adjacent conservatism).
- Eval form: validate `--mode` against `eval_modes` FIRST (cheap vocabulary check, its own rc2 message), then resolve the run dir, then mint `eval/<mode>_<ts>` with `<ts>` = now in `ts_format`.
- **Existing leaf → rc2 refusal** (the `atomic_dir` idiom). This is where F4 prevention lives: a same-second re-mint is always a caller bug, never a false positive — hard-fail is safe here even while the R1 observation gates stay warn.

### 2.5 `omx tree-alias`

`omx tree-alias --name <n> --run <run_spec> [--scope-path <dir>] [--root <r>]`, plus `--list`:

- `--name` must be declared in `links` with `kind: alias` (rc2 otherwise — undeclared aliases are exactly the drift audit would then flag). Target must resolve to a grammar-valid run dir inside the alias's scope (rc2 otherwise; a re-point that would dangle is refused).
- **Alias location when `--scope-path` is omitted**: derived from the target run's own ancestry — the run's enclosing directory at the alias's declared scope level (`scope: root` → the index root). When the scope level is optional and absent on that run's path, rc2 asking for an explicit `--scope-path` (never guess between candidate group dirs).
- Re-point is atomic: create a temp symlink sibling, `os.replace` over the alias name.
- `--list` emits an alias census JSON (name, scope path, target, dangling?).
- Aliases are updated only by explicit calls (exp-loop's skill text mentions `tree-alias --name latest` after a keep; no auto-repoint anywhere).

### 2.6 `omx tree-index`

`omx tree-index [--root <r>] [--check]` — walk the index tree, emit one row per run (run_id · exp · group · created (manifest else ts) · status · eval modes present · analysis count · alias marks) into `<index root>/INDEX.md`, atomic write, sorted newest-first (numeric-safe by ts field).

- The file carries a generated marker (`<!-- generated by omx tree-index; do not hand-edit -->`). **Marker guard:** if an INDEX.md exists *without* the marker (e.g. the reference workspace's hand-written one), refuse with rc2 and require `--adopt` to overwrite — hand-curated indexes are never silently clobbered. The marker guard takes precedence over `--check` (an unmarked file is the no-marker rc2, never a staleness diff).
- `--check`: regenerate in memory, diff against the file, rc2 + diff summary when stale (the same spirit as #25: a generated artifact that can attest its own currency).

### 2.7 `omx clean` (#22 — original design §10.3, honored verbatim)

`omx clean [--scope session|run|all] [--session-id <sid>] [--older-than <Nd>] [--apply] [--root <r>]`:

- **Classify** inside `.omx/` only: KEEP = `profile/` (tree.yaml + seal.json included), `registry/`, `campaigns/`, `runs/*/{results.tsv, ledger.json, decision-log.md}`, `state.json`. SWEEP = `scratch/<sid>/` (scope session; filter by `--session-id`/`--older-than`), `runs/*/cache/` (scope run), orphaned `*.tmp*` (scope all adds both + orphans). `--older-than <Nd>` requires the `d` suffix (rc2 otherwise) and compares against the sweep candidate's top-level directory mtime — session ids embed no reliable timestamp (CLI-supplied ids are arbitrary strings).
- **Dry-run by default**: without `--apply`, print the classification table (path, class, bytes) and touch nothing. With `--apply`, move SWEEP paths to `.omx/.trash/<YYYYMMDD-HHMMSS>/<relpath>` (recoverable; never `rm`). Trash purge is a separate `--purge-trash --i-understand-permanent` double flag — the non-interactive codification of §10.3's "second explicit confirm".
- **Invariants**: the permanent output trees (index/data) are structurally unreachable — clean never leaves `.omx/`. The profile is never swept. `exp-analyze/SKILL.md:406`'s existing `omx clean` reference finally points at a real verb (#25 goes green).

### 2.8 Root resolution ladder (#13)

`omx_core/root.py::resolve_omx_root(explicit=None) -> (Path, stage)` — the audit sketch's 5 stages, in order:

1. `explicit` (`--root`) — always wins.
2. `$OMX_STATE_DIR` set → `$OMX_STATE_DIR/<project_id>` where `project_id = f"{basename}-{sha256(identity)[:16]}"`, identity = the git `origin` remote URL if any, else the absolute toplevel path. Placement/identity split: identity is computed from `git rev-parse --show-toplevel` at cwd — it never climbs submodules; two worktrees of one repo share one identity (the linked-worktree `.omx/` evaporation fix).
3. `.omx-workspace` marker: walk up from cwd, stopping before `$HOME` (exclusive); first directory containing the marker file wins.
4. `git rev-parse --show-toplevel` when inside a repo.
5. cwd.

`OMX_NO_ROOT_LADDER=1` collapses the ladder to explicit-else-cwd (kill-switch family, D9 spirit). Every verb's `--root` becomes optional with default `resolve_omx_root()` — signatures unchanged, purely additive. `omx doctor` gains `{resolved_root, root_stage}` so a surprising resolution is one command away from diagnosable.

### 2.9 Campaign ledger (#28) + probe-novelty extension

`omx_core/campaign.py` + `.omx/campaigns/<campaign_id>/{plan.json, ledger.jsonl}`. `campaign_id` validates with the run_id charset, single segment — it IS the tree's `<group>` segment (D-R2-5; when a project declares a multi-segment group path, campaign_id is the innermost segment, matching the reference usage where `--run_group` is a single name).

- `omx campaign-init --id <g> [--plan <file>] [--root <r>]` — create the dir + `plan.json` (`{campaign_id, created, goal?, baseline_run_id?, notes?}`; only id+created required). rc2 if it exists.
- `omx campaign-log --id <g> --event <launched|kept|discarded|eval|note> [--run <run_id>] [--data <json>] [--root <r>]` — append one jsonl line `{ts, event, run_id?, session_id?, data?}` (O_APPEND single-line write; single-writer assumption documented — the R4 concurrency lock will cover multi-writer). `--data` must parse as a JSON object, rc2 on malformed input (loud-fail, never silent-drop); logging against a non-existent campaign_id is rc2 naming `campaign-init` (explicit-init discipline, symmetric with `campaign-init`'s rc2-on-existing).
- `omx campaign-status --id <g>` — aggregate JSON (event counts, runs seen, last event). `omx campaign-list` — ids + created + event counts.
- Skill wiring: exp-loop step 4 appends keep/discard; exp-design appends the adopted proposal as `note`.
- **probe-novelty extension (D-R2-1, R1 spec 3.10 residue)**: add two scan sources — `campaigns/*/ledger.jsonl` events and `.omx/runs/*/ledger.json` decisions — so the report finally carries the outcome half ("this probe family was tried in <campaign>, outcome discard"). Warn-only nature unchanged (rc0).
